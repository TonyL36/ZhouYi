# 分离部署指南 (Based on Docker Compose)

本指南指导您将前端部署在 **6400** 端口，后端部署在 **6401** 端口。

## 1. 阿里云安全组配置 (必须先做！)

在部署之前，您必须登录阿里云控制台，开放相应的端口。

1.  登录阿里云 ECS 控制台。
2.  找到您的实例 `47.109.193.180`。
3.  点击 **安全组** -> **配置规则**。
4.  点击 **手动添加**，添加以下两条规则：

| 协议类型 | 端口范围 | 授权对象 | 描述 |
| :--- | :--- | :--- | :--- |
| **TCP** | **6400/6400** | **0.0.0.0/0** | 开放前端访问 |
| **TCP** | **6401/6401** | **0.0.0.0/0** | 开放后端API (可选，为了调试或直接调用API) |

> 注意：如果您已经在服务器上运行了其他服务占用这些端口，请先停止它们。

## 2. 部署步骤

### 步骤 1: 上传代码
将整个 `ZhouYi` 文件夹上传到服务器 `/root/` 目录。
```bash
scp -r ZhouYi root@47.109.193.180:/root/
```

### 步骤 2: 启动服务
SSH 登录服务器后，执行以下命令：
```bash
cd /root/ZhouYi
docker-compose up -d --build
```

### 步骤 3: 验证
访问地址：`http://47.109.193.180:6400/book/ZhouYi`

---

## 常见问题
- **无法访问 6400**: 请仔细检查阿里云安全组规则是否生效，以及服务器内部防火墙 (`firewalld` 或 `ufw`) 是否放行了端口。
  - 检查命令: `netstat -tunlp | grep 6400`
- **API Key**: 如需修改，编辑 `docker-compose.yml` 中的 `GLM_API_KEY`。
